Codex Bridge

配置 Codex Bridge，把 ChatGPT 连接到本地项目

跟着这份教程做完，你会得到一个可以粘贴到 ChatGPT 的 MCP 地址。之后你可以让 ChatGPT 查看本地项目、搜索代码、改文件、跑测试、看 diff。

面向正在配置的人。每一步都有“要复制什么”“看到什么算成功”“失败时怎么办”。

首次安装

只需要做一次。以后每天使用时，从“日常启动”开始。

1

打开命令窗口

Mac：按下 Command + 空格，输入 Terminal 或 终端，回车打开。

Windows：点开始菜单，输入 PowerShell，打开 Windows PowerShell。

$ Mac 终端会长这样
PS C:\Users\you> Windows PowerShell 会长这样

复制命令后，粘贴到窗口里，按回车
2

下载安装 Codex Bridge

复制对应系统的命令。脚本会从 GitHub 下载代码到默认目录，安装依赖，并生成本机配置。

Mac M 芯片 / Mac Intel 芯片：

复制到 Mac 终端
curl -fsSL https://raw.githubusercontent.com/baochipham942-eng/codex-bridge/main/install/macos.sh | bash -s -- "$HOME/Projects"

Windows：

复制到 PowerShell
irm https://raw.githubusercontent.com/baochipham942-eng/codex-bridge/main/install/windows.ps1 | iex

默认授权项目目录是 ~/Projects 或 $HOME\Projects。如果你的项目不在这里，先把命令里的项目路径换成真实项目总目录。

3

确认安装成功

安装脚本会在最后运行一次自检，并告诉你日常启动命令。

看到 [OK] Setup finished. 和 [OK] Codex Bridge is ready. 就说明初始化完成。
初始化会检查 Node.js、cloudflared 和 Git。如果脚本提示缺少某个工具，按命令窗口里的提示安装后，再运行同一条命令。Windows 安装后如果命令仍然找不到，关闭 PowerShell，重新打开再试。

日常启动

每次开机后，或服务被关掉后，按这两步启动。

1

启动本地服务

Mac M 芯片 / Mac Intel 芯片：

复制到 Mac 终端
cd "$HOME/Downloads/codex-bridge" && bash scripts/start-local.sh

Windows：

复制到 PowerShell
cd "$HOME\Downloads\codex-bridge"; powershell -ExecutionPolicy Bypass -File scripts\start-local.ps1

成功时会看到：

[OK] Codex Bridge started.
Local MCP URL: http://127.0.0.1:3848/mcp
Health: http://127.0.0.1:3848/health
2

启动公网隧道

ChatGPT 网页端在云上，不能直接访问你的 localhost。这一步会生成一个临时公开地址。

Mac M 芯片 / Mac Intel 芯片：

复制到 Mac 终端
cd "$HOME/Downloads/codex-bridge" && bash scripts/start-cloudflare.sh

Windows：

复制到 PowerShell
cd "$HOME\Downloads\codex-bridge"; powershell -ExecutionPolicy Bypass -File scripts\start-cloudflare.ps1

成功时会看到：

[OK] Cloudflare tunnel started.
MCP URL: 复制这里显示的实际地址，如：https://abc-123.trycloudflare.com/mcp
Use this URL in ChatGPT Web.
每次重新启动隧道，trycloudflare.com 地址可能会变。地址变了以后，需要回 ChatGPT 连接器里更新 URL。

在 ChatGPT 网页端创建连接器

打开 ChatGPT 网页端，找到 应用 / Apps / Connectors / 连接器 相关入口，创建一个自定义 MCP 连接器。

表单项填写内容
名称Codex Bridge
描述Connect ChatGPT to local code, tests, and diffs through MCP.
服务器 URL填入 启动公网隧道 那一步输出的 MCP URL
身份验证第一次使用选 无 / No authentication
如果页面提示自定义 MCP 有风险，需要勾选确认后再创建。这个工具会访问你允许的本地项目目录，不要把 tunnel URL 发到公开群。
Create connectorServer URL
Codex Bridge
粘贴命令窗口输出的 MCP URL
No authenticationCreate
看不到创建入口时，先检查 ChatGPT 设置里是否有 Developer mode。公司工作区可能需要先申请自定义连接器权限。

ChatGPT 自定义 MCP 入口可能随产品更新调整。遇到 UI 不一致时，以 ChatGPT 的 Connectors / Apps / Developer mode 相关设置为准。

测试是否连通

新开一个 ChatGPT 对话，选择刚创建的 Codex Bridge 连接器，然后发送：

复制到 ChatGPT
请通过 Codex Bridge 调用 bridge.status，告诉我当前服务是否正常。
如果 ChatGPT 回复里出现 codex-bridge、allowedProjectRoots、tools 等信息，就说明连接成功。

之后可以试这个：

复制到 ChatGPT
请通过 Codex Bridge inspect 我的项目，然后给我一份结构摘要。项目路径是：把这里换成你的真实项目路径

安全边界

Codex Bridge 默认只开放一个你指定的项目总目录，并拒绝常见密钥文件。

  • 授权目录写在 bridge.policy.local.json 的 allowedProjectRoots 里。
  • 拒绝读取的文件写在 denyGlobs 里。
  • 脚本命令会过滤 sudo、危险删除命令和 curl | sh 这类模式。
tunnel URL 相当于临时入口。只在当前配置期间使用，不要贴到公开频道。

停止服务

不用时可以停止本地服务和 tunnel。

Mac M 芯片 / Mac Intel 芯片：

复制到 Mac 终端
cd "$HOME/Downloads/codex-bridge" && bash scripts/stop-local.sh

Windows：

复制到 PowerShell
cd "$HOME\Downloads\codex-bridge"; powershell -ExecutionPolicy Bypass -File scripts\stop-local.ps1

常见问题自查

遇到问题先运行 doctor，它会检查 Node、cloudflared、Git、本地服务、MCP 工具列表和最近的 tunnel 地址。

Mac M 芯片 / Mac Intel 芯片：

复制到 Mac 终端
cd "$HOME/Downloads/codex-bridge" && bash scripts/doctor.sh

Windows：

复制到 PowerShell
cd "$HOME\Downloads\codex-bridge"; powershell -ExecutionPolicy Bypass -File scripts\doctor.ps1
现象处理
ChatGPT 创建连接器时报错确认 URL 以 /mcp 结尾，确认 tunnel 脚本仍在运行，重新复制最新 URL。
ChatGPT 能看到连接器，但调用工具失败先运行对应系统的 doctor 脚本。health 失败时，重新运行本地启动脚本。
提示端口 3848 被占用先运行停止脚本。还不行时，Mac 用 CODEX_BRIDGE_PORT=3850 bash scripts/start-local.sh，Windows 用 $env:CODEX_BRIDGE_PORT=3850; powershell -ExecutionPolicy Bypass -File scripts\start-local.ps1。
提示 Path outside allowed roots项目目录不在授权范围内。重新运行对应系统的 bootstrap 脚本，把项目总目录填进去。
读取 .env 或私钥失败这是正常保护。默认拒绝读取 .env、私钥、.npmrc、.ssh。
